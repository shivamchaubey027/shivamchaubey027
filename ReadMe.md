# Building Reliable Infrastructure at Scale

I design backend systems that don't break at 3 AM.

Currently building a **distributed rate limiter** that handles race conditions, network partitions, and Redis failures across multiple instances. Real distributed systems, real production concerns.

## What I Build
- **Fault-tolerant systems**: Multi-region disaster recovery (RPO < 5min, RTO < 15min)
- **Kubernetes tooling**: Real-time resource monitors that track cost, not just metrics
- **Infrastructure automation**: Terraform modules that teams can actually use

## My Engineering Principles
- Systems should fail gracefully, not catastrophically
- Observability isn't optional - if you can't measure it, you can't fix it
- Infrastructure is code, and code should be reviewed, tested, and versioned

**Currently exploring**: Distributed consensus algorithms, Kubernetes controllers, and why Redis Lua scripts are your friend in distributed systems.
